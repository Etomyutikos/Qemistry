Do (queue)
Executes a single queue or all queues When called with an argument, it calls that Queue's Do function.

When called without an argument, it calls all Queues' Do functions.

Parameters:

  • queue string The name of the Queue to be executed.
Delete (queue)
Removes a Queue from the master list.

Parameters:

  • queue string The name of the Queue to be deleted.
Queue (name, conditions[, options])
Queue constructor.

Parameters:

  • name string The name of the new Queue.
  • conditions table, string, bool or function The conditions upon which this Queue operates (ex. Balance, Equilibrium). This may be a table of strings or functions. This may be a string, a function, or true. Functions must ultimately return a boolean value. Strings must be the "address" of a boolean value ("my_system.balance").
  • options table A table of strings or a string. Values must be one of the valid_options.

Returns:

    Queue
queue:Add (action)
Adds an Action to the Queue.

Parameters:

  • action string, function or table If it is a string, it assumes the Queue's Conditions. That string will be passed to the MuD via send().

    If it is a function, it assumes the Queue's Conditions. That function will be called directly.

    If it is a table, that must be a list of strings or functions, or a formatted Action table. The strings and functions behave as above. The formatted table must contain the field "code", and optionally one or both of "required" and "consumed". "required" and "consumed" function as do Queue Conditions. The "code" functions as a non-formatted action above.
